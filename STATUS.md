## Описание
###### Description
---
Трекер дел - программа, объединяющая возможности приложений для ведения списков дел, календарей и трекеров задач. Позволяет легко и быстро организовать свое время, не беспокоясь о возможности забыть выполнить что-нибудь необходимое. С помощью данного приложения пользователю не придется запоминать все множество рутинных дел, а предоставить эту задачу программе, которая в удобной форме организует и оптимизирует ваш распорядок. Программа позволит избавиться от множества календарей и планировщиков, тем самым собрав все необходимые функции в одном приложении.
Программа будет написана на языке программирования Python.

3) _Яндекс Трекер_  
Для работы с приложением требуется наличие аккаунта и необходимо купить один из предложенных тарифов в соответствии с числом пользователей компании.

Для каждой задачи в Трекере заводится своя страница — с описанием, ответственными, сроками выполнения, ссылками на связанные задачи и возможностью запрашивать необходимы данные и комментировать детали, согласовывать результаты и уведомлять заинтересованных. Для добавления однотипных задач можно создавать шаблоны. Можно создавать напоминания и призывы, как для себя так и для других сотрудников.

Задачи разных отделов и команд группируются по отдельным очередям, для которых можно установить параметры задач в соответствии с методологией, выбранной командой: Базовый, Поддержка - для задач отдела техподдержки, Подтверждения - для процессов, требующих подтверждения(задачи отдела кадров и др.), Scrum - вместе с оцередью создается доска задач, Kanban, Разработка(базовый) - для задач на разработку ПО, Разрабтка(продвинутый) - дополнительно включает этап тестирования либо создать свою методологию. Можно сделать копию очередей.

Сущестуют фильтры для удобности отображения и группировки задач в списке, по приоритету, по версии, по исполнителю, по статусу, по дедлайну и другие.

Для большей наглядности можно сделать дашборды, на которых руководитель может собрать инфрмацию о работе всех команд, участвующих в проекте, и исполнитель - список своих задач со сроками и приоритетами.

Поддержка досок Agile, состоящих из колонок, отражающих стадию выполнения задачи или другое, между которыми можно переносить задачи.

__Плюсы приложения__:  
1. Большое число характеристик задачи  
2. Информативные дашборды  
3. Очереди для задач в соответствии с их типом  
4. Учет времени и трудозатрат  
__Минусы приложения__:  
1. Работа на платной основе.  
2. Нет приложения для системы Android.  
__Полезные возможности__:  
1. По такому же принципу можно организоваь дашборды  
2. Система очередей  
3. Доски Agile.  

## Сценарий использования
###### Usage scenario
---
Пользователь при помощи данного приложения может:  
1. Запланировать например, уборку по дому на субботу, в качестве дел здесь будут, к примеру: подмести и помыть пол, вытереть пыль, помыть посуду и другое. Можно установить напоминание за день до дел, чтобы не забыть. Если не выполнить дела, то задание помечается как проваленное по истечению дня, на которое оно было запланировано.  
2. Создать группу однотипных списков дел, например, покупка товаров в различных видах магазинов. Можно скопировать списки в другую группу однотипных задач.
3. Создать план, который автоматически создаст задачу "сходить на секцию по плаванию в субботу".  
4. Например организовать работу по проведению свадьбы, где для каждого сотрудника установить отдельный вид работ: кто-то украшает зал, кто-то занимается приготовлением пищи, кто-то будет тамадой. Для каждого вида таких работ организатор может добавить уточнения, сроки, условия выпонения и следить за правильностью исполнения.  
5. Создавать событие, например экзамен, указав его дату, а в описании события написать ФИО преводователя, место экзаменации. После чего событие отобразиться в календаре.  
6. В качестве лидера команды разработки, допустим игры, пользователь может распределить каждую отдельную задачу отдельным специалистам, например за создание концепт-артов отвечают художники, за программирование бизнес-логики -  программисты, за сценарий - сценаристы и т.д. После этого каждую конкретную задачу можно разбить на множество подзадач, каждую из которых распределяем по конкретным специалистам, установив перед этим зависимости между подзадачами. При необходимости, удобно будет поставить человека на другую подзадачу. По мере выполнения отдельных подзадач и задач, будет отображаться прогресс выполнение проекта. Можно установить сроки выполнения конкретных подзадач, чтобы специалисты могли взвешенно организовать свою работу. Обсуждение задачи поможет направлять процесс создания игры в правильном направлении и избежать ошибок. При необходимости, в каждую подзадачу можно вносить изменения и уточнения.  
7. Поставить, к примеру, себе цель проходить определенное растояние каждый день. При этом будет виден прогресс выполнения цели в день, месяц.

## Планируемые возможности
###### Capabilities
---
Согласно описанным выше сценариям планируется, что программа будет способна реализовать следующие возможности пользователя:  
1. Вести список дел, изменять, добавлять, удалять, находить дела по названию; создавать напоминания; получать уведомления о провале в случае, если пользователь не смог выполнить дело в заданные сроки, добавлять описание дела, приглашать других участников для совместного выполнения дела, просматривать списки выполненных дел, воспользоваться опцией карма (проследить как успешно выполняются дела).    
2. Создавать план, который будет автоматически организовавыть дела в определенное время суток, дни недели, или в определнный месяц, с напоминанием о необходимости выполнения за определенное время до начала дела.  
3. Создавать задачи, разбивая их на подзадачи, назначая ответственных за выполнение каждой подзадачи; устанавливать сроки выполнения подзадачи, управлять системой принятия задач и подзадач к исполнению, следить за готовностью и прогрессом выполнения каждой подзадачи, фиксировать все необходимые для выполнения задач и подзадач условия.  
4. Создавать события, отмечать их на календаре событий, приглашать на них других участников, устанавливать время и место события.  
5. Создавать проекты, в которых будет корневая задача, основные крупные части, каждая из которых будет разбита на иерархию подзадач, которые так же могут быть разбиты на подзадачи; редактировать и отслеживать дерево составного дела; распределять подзадачи по отдельным людям; определить сроки выполнения; стадию выполнения (начал, в процессе, готово); устанавливать связи между делами (отмечать, что какая-нибудь подзадача блокирует выполнение другой подзадачи или требует выполнение совместно с другой подзадачей); иметь возможность отслеживать ход выполнения и при необходимости давать указания, уточнения, перераспределять участников, выполнивших свою подзадачу на другую подзадачу, находящуюся на стадии выполнения; создавать и просматривать дорожную карту проекта (сценарий выполнения); организовать систему уведомлении о приближении сроков сдачи проекта заказчику; получать уведомления о изменении своих и чужих отслеживаемых дел, оповещения об успешном выполении проекта, о закрытии всех подзадач и задач.  
6. Создавать и редактировать цели, устанавливать сроки к которым необходимо этой цели достичь, устанавливать удобное время, выбирать оптимальный вариант достижения цели, просматривать статистику цели.  

## Логическая архитектура
###### Logical architecture
---
_Структуры данных имеют предварительное описание, в ходе разработки программы они могут неоднократно уточняться и редактироваться._  
1. Списки дел можно хранить с помощью структуры данных массив, имеющей уже реализованные методы добавления, удаления, поиска и редактирования. Каждое дело представляется как объект класса Задача имеющее поля: описание, название, автор, дата, является ли избранным, статус, участники дела. После того как дело выполнено, оно удаляется из списка запланированных дел и помещается в список выполненных дел, который так же хранится в массиве. Опцию карма можно реализовать как отношение количества выполненных дел к количеству запланированных. Для каждого дела можно включить опцию напоминания.
2. План реализовать при помощи объекта класса План, который будет содержать функцию создания дел в соответствии с данными о необходимом времени суток для выполнения, напоминанием о необходимости выполнения. Если планов несколько то их так же следует помещать в массив объектов класса План.  
3. Задача представляет собой объект класса Задача, содержащий поля: подзадачи(который так же можно определить как массив объектов класса Задача), ответственный за выполнение задачи/подзадачи, сроки выполнения, принята ли задача к исполнению, прогресс выполнения задач/подзадач, условия задач/подзадач и прикрипленные материалы, описание, условие выполнения.  
4. Событие представляет собой объект класса Задача, содержащий поля: дата события, длительность события, место проведения, напоминание о событии, участники события.   
5. Проект представляет собой объект класса Задача, который содержит поля: задачи(массив объектов класса Задача, каждый элемент которого может содержать свои подзадачи), прикрепленные материалы, сроки выполнения, стадия выполнения(начал, в процессе, готово), условия выполнения задачи(совместно с какой-нибудь другой задачей или без нее), описание проекта, дорожная карта проекта, напоминания о приближении дедлайна; функции: перераспределние участников между задачами, отслеживание дел, оповещение о выполнении проекта, закрытий задач и подзадач, и другие.  
6. Цель представляет собой объект класса Задача, имеющие поля: сроки выполнения, требуемое значение параметра цели, текущее значение величины параметра цели, прогресс за текущий день, прогресс по предыдущим дням (месяцам), составляющим статистику цели; содержит функцию, которая находит удобное время для выполнения цели и отображает это время как подсказку.  
_Позднее все описанные данные будут храниться в базе данных (на данный момент с базами данных я работать не умею)_.  

## Этапы разработки
###### Stages of development
---
В процессе разработки, планируется выпустить следующие версии программы:  
1. Прототип. Планируется, что он будет поддерживать как минимум работу со списками дел в интерфейсе командной строки, как максимум иметь реализацию всех необходимых для работы программы классов.  
2. Базовая версия. В данную версию будет входить возможность поддержки следующих возможностей:  объединие списков дел в группы, дублирование списков, создавние планов, задач и подзадач, создание событий - это задача-максимум. Задача-минимум: обеспечить поддержу приложением объединения списков дел в группы, дублированием списков, созданием планов.  
3. Расширенная версия с поддержкой базы данных. Представляет собой усложнение базовой версии путем добавления организации проектов, досок руководителя, целей и поддержки базы данных - задача-максимум. Задача-минимум - полностью готовая базовая версия включающая дополнительно поддежку базы данных и позволяющая работать с организацией проектов.  
4. Веб-версия. Создание веб-интерфейса для расширенной версии программы, дополнение, тестирование, исправление ошибок предыдущих версий - задача-максимум. Задача-минимум - получение веб-версии программы, с урезанными возможностями (некоторая планируемая функциональность может быть не реализована, например доски руководителя и цели).
