## Описание
###### Description
---
Трекер дел - программа, объединяющая возможности приложений для ведения списков дел, календарей и трекеров задач. Позволяет легко и быстро организовать свое время, не беспокоясь о возможности забыть выполнить что-нибудь необходимое. С помощью данного приложения пользователю не придется запоминать все множество рутинных дел, а предоставить эту задачу программе, которая в удобной форме организует и оптимизирует ваш распорядок. Программа позволит избавиться от множества календарей и планировщиков, тем самым собрав все необходимые функции в одном приложении.
Программа будет написана на языке программирования Python.

**Трекеры задач**:  
1) RedminePM  
Приложения требует наличия аккаунта. Рассмотрим возможности приложения.

В главном меню приложения реализованы 4 вкладки:  
1. Issue  
2. My task  
3. Search  
4. Setting  
На вкладке "Issue" размещены проекты. При выборе проекты, отображается список всех задач проекта. Если выбрать какую-нибудь задачу из списка, появиться полная информация о задаче, из которой можно узнать следующее: Назввание проекта, статус, приоритет, ответственное лицо, процент выполнения, комментарии, трекер, приоритет, к какому проекту принадлежит, дату начала, срок выполнения, версию, автора, дату создания, последнее обновление, расчетное время выполнения, категорию, страну, пользователя, пользовательский текст, а так же отслеживание изменений в информации. Каждый пункт (кроме автора) можно изменять по мере необходимости. Для каждой группы задач реализован фильтр задач (открытая, закрытая задача), сортировка по id, срокам выполнения, дате начала выполнения, последнему обновлению, приоритету, и для сортировки задавать вывод по возрастанию или убыванию.

На вкладке "My task" содержатся все задачи пользователя, которые также можно редактировать.

Вкладка "Search" позволяет найти проекты и задачи по названию, организавать фильтрованный поиск задач проекта по статусу, по трекеру, ответственному лицу, приоритету и так далее. Есть дорожная карта, показывающая сценарий выполнения задач. WIKI содержит информацию о проектах.

На вкладке "Setting" можно изменить настройки аккаунта или переключиться на другой.

__Плюсы__ приложения:  
1. Удобный поиск задач с возможностью установки фильтра и сортировкой.  
2. Удобная система редактирования задач.  
3. Наличие дорожной карты проекта.
4. Возможность использования сразу нескольких аккаунтов.  
5. Присоедние файлов к задаче.  
__Минусы__:  
1. Требует подключение к интернету.  
2. Не очень приятный и устаревший интерфейс.

Приложение требует наличие аккаунта. Так же требуется купить пакет для доступа к продукту.
Приложение имеет ряд вкладок:
1. Issues  
2. Projects  
3. Notification  
4. Account
На вкладке "Issues" показаны выполненные задачи проектов и есть возможность создания задачи. Так же есть опция показа задач по категориям: предназначенные пользователю, задачи, о которых сообщили пользователю, недавно просмотренные задачи, наблюдаемые задачи, все, открытые, выполненные, недавно созданные, недавно решенные, недавно обновленные задачи. На вкладке "Projects" показаны все созданные проекты. Здесь можно добавить новый проект, указав его название, и один из шаблонов: Agility - упрощенный вид проекта, подходит для новичков, Project Managment - управление деятельностью бизнесс-проекта, используется для проектов привязанных ко времени или к результатам выполнения, Task tracking - организация задач для команды или для одного человека в рамках проекта, его можно рассмтривать как длинный список задач, Process control - отслеживает деятельнисть и задачи для повторяющихся процессов, подходит для организации задач, которые имеют прогресс выполнения. Выбрав один из проектов можно создать для него задачу, указав резюме, описание, можно прикрепить документ или фотографию, определить ответственное лицо, приоритет, сроки выполнения.

На вкладке "Notifications" отображается активность пользователей.

На вкладке "Account" содержатся настройки и информцая о пользовательском аккаунте.

3) Яндекс Трекер  
Для работы с приложением требуется наличие аккаунта и необходимо купить один из предложенных тарифов в соответствии с числом пользователей компании.

Для каждой задачи в Трекере заводится своя страница — с описанием, ответственными, сроками выполнения, ссылками на связанные задачи и возможностью запрашивать необходимы данные и комментировать детали, согласовывать результаты и уведомлять заинтересованных. Для добавления однотипных задач можно создавать шаблоны. Можно создавать напоминания и призывы, как для себя так и для других сотрудников.

Задачи разных отделов и команд группируются по отдельным очередям, для которых можно установить параметры задач в соответствии с методологией, выбранной командой: Базовый, Поддержка - для задач отдела техподдержки, Подтверждения - для процессов, требующих подтверждения(задачи отдела кадров и др.), Scrum - вместе с оцередью создается доска задач, Kanban, Разработка(базовый) - для задач на разработку ПО, Разрабтка(продвинутый) - дополнительно включает этап тестирования либо создать свою методологию. Можно сделать копию очередей.

Сущестуют фильтры для удобности отображения и группировки задач в списке, по приоритету, по версии, по исполнителю, по статусу, по дедлайну и другие.

Для большей наглядности можно сделать дашборды, на которых руководитель может собрать инфрмацию о работе всех команд, участвующих в проекте, и исполнитель - список своих задач со сроками и приоритетами.

Поддержка досок Agile, состоящих из колонок, отражающих стадию выполнения задачи или другое, между которыми можно переносить задачи.
